Stubz Documentation

A guideline to Stubz' endpoints. This web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Business Logic URL & Query Parameters HTTP Method Request format Response format
Return a list of all movies /movies GET None A JSON object holding all movies along with their info
Return data about a single movie by title /movies/[title] or /movies/:title GET None A JSON object holding data about a movie:

],

"Featured": true,
"Year": "2012"
}
Return movies according to the genre /movies/?genre=[genre] or /movies/?genre=:genre GET The search query contains a genre type A JSON object holding all movies that share the genre
Return data about a director by name /directors/[name] or /directors/:name GET None A JSON object holding data about the director and associated movie(s)
Return a list of all users /users GET None A JSON object holding all users along with their info

Allow users to update their info /users/username or /users/:username PUT A JSON object holding the update about the user:
{
"Birthday": "2000-01-02"
} 		>A verification text stating whether or not the update has been successful
Allow users to add movies to their list of favorites /users/username/movies/[movie] or /users/:username/movies/:movie POST Registered user individual adds a movie to their list of favorites A verification text stating whether or not the movie has been successfully added
Allow users to remove movies from their list of favorites /users/username/movies/[movie] or /users/:username/movies/:movie DELETE Registered user individual removes a movie from their list of favorites A verification text stating whether or not the movie has been successfully deleted
Allow existing users to deregister /users/username or /users/:username DELETE None A verification message stating whether or not the user was able to successfully deregister
